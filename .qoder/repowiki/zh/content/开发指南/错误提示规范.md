# 错误提示文案规范

本文档定义了系统中各类场景下的标准错误提示文案，旨在提供一致、友好且安全的体验。

## 1. 登录与认证

### 1.1 账号密码登录
| 场景 | 错误原因 | 标准提示文案 | 备注 |
|---|---|---|---|
| 表单校验 | 邮箱为空 | 请输入登录邮箱 | |
| 表单校验 | 邮箱格式错误 | 请输入有效的邮箱地址 | |
| 表单校验 | 密码为空 | 请输入密码 | |
| 登录失败 | 账号或密码错误 | 账号或密码错误，请重试 | 避免提示"账号不存在"以防枚举 |
| 登录失败 | 账号被禁用 | 账号已被停用，请联系管理员 | |
| 登录失败 | 密码未设置 | 账号未激活或密码未设置 | |
| 登录失败 | 连续失败多次 | 尝试次数过多，请稍后再试 | |

### 1.2 2FA (双因素认证)
| 场景 | 错误原因 | 标准提示文案 | 备注 |
|---|---|---|---|
| 校验 | 验证码为空 | 请输入 Google 验证码 | |
| 校验 | 格式错误 | 请输入 6 位数字验证码 | |
| 验证失败 | 验证码错误 | 验证码错误或已失效，请重试 | |
| 绑定 | 密钥丢失 | 2FA 设备丢失？请使用备用码或联系管理员 | |

### 1.3 密码重置与激活
| 场景 | 错误原因 | 标准提示文案 | 备注 |
|---|---|---|---|
| 链接失效 | Token 过期/无效 | 链接已失效，请重新请求 | |
| 重置限制 | 频率限制 |请求过于频繁，请稍后再试 | |

## 2. 通用交互

### 2.1 网络与系统
| 场景 | 错误原因 | 标准提示文案 | 备注 |
|---|---|---|---|
| 网络请求 | 断网/超时 | 网络连接失败，请检查您的网络 | |
| 服务器错误 | 500/502 | 系统服务暂时不可用，请联系技术支持 | |
| 权限不足 | 403 | 您没有权限执行此操作 | |

## 3. 开发建议

1. **前端捕获**: 尽量在前端进行基础格式校验，即时反馈。
2. **后端透传**: 对于业务逻辑错误（如库存不足），后端返回明确的 `code`，前端根据 `code` 映射文案，尽量避免直接展示后端原始报错信息（`error.message`），除非该 Message 经过了国际化/标准化处理。
3. **安全隐私**: 登录类接口严禁返回 "该邮箱未注册" 等可用于撞库的提示。
